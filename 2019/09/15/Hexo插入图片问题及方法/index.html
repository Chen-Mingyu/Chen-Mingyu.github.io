Hexo插入图片问题及方法

发表于 2019-09-15 分类于 搭建博客 阅读次数：

前言

搭建好博客后，迫不及待写了几篇博客，发现使用hexo在博客中插入图片确实是个问题，例如：生成的图片路径格式不对，路径对但是显示失败

插入图片方法

本方法借助hexo-asset-image插件，将本地图片上传

  1. 将博客本地文件夹中的“_config.yml”中的“post_asset_folder”改为“true”
  2. 打开“命令提示符”，切换到博客目录下，执行“npm install hexo-asset-image –save”安装插件，安装完成后，使用“hexo n 标题名称”生成md博文的同时，会生成与标题名称同名的文件夹，用来存放图片
  3. 在博文中引用图片时，先将图片放到该文件夹中，再使用“”引用图片。PS：实测使用“”也可正常显示图片

遇到问题

出现“.com//test.jpg”

进入“public\2019\09\12\”，打开“index.html”，查看“img src=”xxx””，如果“xxx”为“.com//test.jpg”

  1. 更改博客目录下的“_config.yml”的“url”，将超链接地址改为你的Github仓库地址，如：“https://xxxx.github.io/”
  2. 打开“/node_modules/hexo-asset-image/index.js”，用下面代码替换，index.js，保存后，命令提示符执行“hexo clean && hexo g”，查看“public\2019\09\12index.html”的“<img src=”xxx””>，看“xxx”是否变为“2019\09\12\xxx\test.jpg”

图片地址正常，但图片显示失败

如果确认“/node_modules/hexo-asset-image/index.js”的地址正确，即与上述解决方案结果相同，但是图片显示失败，可以尝试修改博客目录下的“pakage.json”的“hexo-asset-image”，改为“0.0.1”，即修改版本，改为0.0.1版本，在命令提示符执行“hexo clean && hexo g && hexo s”在线预览，看能否正常显示
